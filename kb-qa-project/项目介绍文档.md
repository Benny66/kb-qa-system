# 📚 基于大模型的知识库智能问答系统

> 毕业设计 / 课程设计项目 · 前后端完整源码 · 开箱即用

---

## 🎯 项目简介

本系统是一个**基于大语言模型（LLM）的知识库智能问答平台**，用户可以将自己的文档资料上传为知识库，系统通过 AI 大模型对知识库内容进行理解，实现精准的自然语言问答。

项目采用**前后端分离**架构，接口设计规范，代码结构清晰，注释完整，适合作为毕业设计、课程设计或学习参考使用。

## ✨ 核心功能

### 🔐 用户登录
- 账号密码登录，JWT Token 鉴权
- 前端路由守卫，未登录自动跳转
- Token 持久化存储，刷新不掉线

### 📂 知识库管理
- 支持上传 TXT 格式文本文件作为知识库
- 自动统计文件大小、字符数量
- 支持删除知识库（同步清理关联问答历史）
- 文件安全存储，UUID 命名防冲突

### 💬 AI 智能问答
- 选择知识库后即可开始对话
- AI 严格基于知识库内容作答，不编造信息
- 知识库内容不存在时明确告知用户
- 实时显示 Token 消耗量
- 支持多轮对话展示，消息气泡 UI

### 📋 问答历史
- 完整保存每次问答记录
- 支持按知识库筛选历史
- 分页加载，支持展开/收起长文本
- 支持删除单条历史记录

## 📡 接口设计（RESTful）

| 方法 | 接口路径 | 功能说明 |
|------|----------|----------|
| POST | `/api/auth/login` | 用户登录，返回 JWT Token |
| GET | `/api/auth/me` | 获取当前用户信息 |
| GET | `/api/kb` | 获取知识库列表 |
| POST | `/api/kb/upload` | 上传 TXT 知识库文件 |
| DELETE | `/api/kb/:id` | 删除知识库 |
| POST | `/api/chat` | 发起 AI 问答 |
| GET | `/api/chat/history` | 获取问答历史（分页） |
| DELETE | `/api/chat/history/:id` | 删除单条历史 |
| GET | `/api/health` | 服务健康检查 |

统一响应格式：
```json
{
  "code": 200,
  "msg": "success",
  "data": { ... }
}
```

## 🗄️ 数据库设计

### users（用户表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| username | VARCHAR(64) | 用户名（唯一） |
| password_hash | VARCHAR(256) | Werkzeug 哈希密码 |
| created_at | DATETIME | 创建时间 |

### knowledge_bases（知识库表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| user_id | INTEGER | 外键 → users |
| name | VARCHAR(128) | 知识库名称 |
| filename | VARCHAR(256) | 服务器存储文件名 |
| file_path | VARCHAR(512) | 文件完整路径 |
| file_size | INTEGER | 文件大小（字节） |
| char_count | INTEGER | 字符数 |
| created_at | DATETIME | 上传时间 |

### chat_histories（问答历史表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| user_id | INTEGER | 外键 → users |
| kb_id | INTEGER | 外键 → knowledge_bases |
| question | TEXT | 用户提问 |
| answer | TEXT | AI 回答 |
| tokens_used | INTEGER | 消耗 Token 数 |
| created_at | DATETIME | 问答时间 |

## 🚀 运行方式

### 环境要求
- Python 3.11+
- Node.js 18+
- 智谱 AI API Key（[免费申请](https://open.bigmodel.cn/usercenter/apikeys)，注册即送额度）

### 后端启动
```bash
cd kb-qa-backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env        # 填入 ZHIPUAI_API_KEY
python app.py               # 启动在 http://localhost:5001
```

### 前端启动
```bash
cd kb-qa-frontend
npm install
npm run dev                 # 启动在 http://localhost:5173
```

## 💡 系统亮点

1. **知识库隔离**：每个用户的知识库和问答历史完全隔离，数据安全
2. **Prompt 工程**：将知识库全文注入 System Prompt，AI 回答严格基于文档内容，不产生幻觉
3. **编码兼容**：自动识别 UTF-8 / GBK 编码的 TXT 文件，兼容中文文档
4. **接口规范**：统一的 RESTful 接口设计，统一响应格式，前后端解耦
5. **安全设计**：密码 Hash 存储、JWT 鉴权、文件名 UUID 化、上传类型校验
6. **工程规范**：`.env` 环境变量隔离、`.gitignore` 保护敏感文件、代码注释完整
